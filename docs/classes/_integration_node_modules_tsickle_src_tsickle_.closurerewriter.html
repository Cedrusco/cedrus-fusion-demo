Class ClosureRewriter

A Rewriter subclass that adds Tsickle-specific (Closure translation) functionality.

One Rewriter subclass manages .ts => .ts+Closure translation. Another Rewriter subclass manages .ts => externs translation.

symbolsToAliasedNames

symbolsToAliasedNames: Map<Symbol, string> = new Map<ts.Symbol, string>()
  • Defined in integration/node_modules/tsickle/src/tsickle.ts:161

A mapping of aliases for symbols in the current file, used when emitting types. TypeScript emits imported symbols with unpredictable prefixes. To generate correct type annotations, tsickle creates its own aliases for types, and registers them in this map (see emitImportDeclaration and forwardDeclare() below). The aliases are then used when emitting types.

emitFunctionType

  • emitFunctionType(fnDecls: SignatureDeclaration[], extraTags?: Tag[]): string[]
    • Defined in integration/node_modules/tsickle/src/tsickle.ts:178

    Handles emittng the jsdoc for methods, including overloads. If overloaded, merges the signatures in the list of SignatureDeclarations into a single jsdoc.

    • Total number of parameters will be the maximum count found across all variants.
    • Different names at the same parameter index will be joined with "or"
    • Variable args (...type[] in TypeScript) will be output as "...type", except if found at the same index as another argument.

    Parameters

    • fnDecls: SignatureDeclaration[]

      Pass > 1 declaration for overloads of same name

    • Default value extraTags: Tag[] = []

    Returns string[]

    The list of parameter names that should be used to emit the actual function statement; for overloads, name will have been merged.
